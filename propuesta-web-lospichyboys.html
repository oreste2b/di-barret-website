01 — Tesis

El producto cambió. El sitio no.

Hace 10 años un creador podía operar desde un sitio template. Hoy un podcast top con director profesional, partnership con Sonoro y 20+ marcas por episodio es una productora. El sitio debe operar como tal — no como un perfil.

  1. El sitio es el sales-floor de la publicidad.

    El motor #1 del negocio (~60% revenue) son las integraciones de marca dentro del podcast. Hoy ese motor depende de un mailto. Va a depender de una infra seria.

  2. El sitio es el box-office de los live shows.

    El motor #2 (~25%) son las obras y podcasts en vivo. Hoy: "Tampa muy pronto" sin fecha. Va a ser: calendario en tiempo real con ticketing y waitlist por ciudad.

  3. El sitio es el on-ramp de Rebeca.

    Cada mes sin landing de la película = 5-10k emails de super-fans no capturados. Ese fan-base es lo que decide si la película hace 1M o 10M el primer fin de semana.

  4. El sitio es la prueba de que son una marca, no un perfil.

    Wix con copyright caducado comunica "perfil personal". Site custom con animaciones, scroll-trigger y media kit profesional comunica "productora de cine y comedia".

04 — Stack técnico

Custom build. Cero compromisos.

El stack que usa Vercel internamente para su propio site, Linear, Resend, Cal.com. Next.js 16 con scroll cinematográfico (GSAP + Lenis), audio embebido, CMS headless para que el equipo Pichy publique sin tocar código.

Layer 01 · Frontend

Next.js 16 + React Server Components

App Router, Server Components donde no hace falta JS (mejor SEO + velocidad), Cache Components con PPR para revalidate selectivo. Hosteado en Vercel — Fluid Compute, edge cache global, deploys atómicos.

  • Next.js 16 App Router
  • React Server Components
  • Vercel Fluid Compute
  • Edge cache global
Layer 02 · Animación

GSAP + ScrollTrigger + Lenis

GSAP para timelines complejos, ScrollTrigger para scroll-driven sequences (hero, transiciones, parallax). Lenis para smooth scroll suave (60fps en todo). Three.js opcional para hero con shader o WebGL particle field.

  • GSAP 3 + ScrollTrigger
  • Lenis smooth scroll
  • Three.js (hero opcional)
  • prefers-reduced-motion respected
Layer 03 · Datos & contenido

Sanity CMS + Sonoro API + Stripe

Sanity CMS para que el equipo Pichy publique episodios, sponsors, eventos sin tocar código. Sonoro / RSS feed para sync automático del podcast. Stripe + Memberful para Pichy Club. ConvertKit / Beehiiv para email.

  • Sanity headless CMS
  • Sonoro / RSS sync
  • Stripe + Memberful (Club)
  • ConvertKit (email funnel)
